\section{Постановка задачи}
\label{sec:Chapter1} \index{Chapter1}

\subsection{Описание задачи}
\label{subsec:task-description}

Рассматривается задача повышения качества многошаговых рассуждений (multi-hop reasoning) больших языковых моделей посредством использования явной графовой структуры как для оценки, так и для обучения.

Пусть дан вопрос $q$, ответ на который требует последовательного логического вывода через $k$ промежуточных шагов. В графе знаний $\mathcal{G} = (V, E)$, где $V$~--- множество сущностей, $E \subseteq V \times R \times V$~--- множество рёбер с метками отношений из $R$, для вопроса~$q$ выделяется \textit{эталонный подграф} $G^* = (V^*, E^*)$, описывающий корректную цепочку вывода от исходных фактов к финальному ответу.

Модель $\mathcal{M}$, получая на вход вопрос $q$, генерирует текстовый ответ $a$, содержащий цепочку рассуждений. Текстовый ответ преобразуется в \textit{сгенерированный граф} $\hat{G} = (\hat{V}, \hat{E})$ с помощью автоматизированного пайплайна извлечения троек.

Задача состоит в:
\begin{enumerate}
    \item разработке методов автоматического преобразования текстовых рассуждений в графовое представление и сопоставления с эталонными графами;
    \item определении и формализации метрик структурного соответствия между $\hat{G}$ и $G^*$;
    \item использовании указанных метрик в качестве функции награды при дообучении модели методами обучения с подкреплением;
    \item экспериментальной проверке гипотезы о том, что графовая награда повышает качество рассуждений по сравнению с бинарной оценкой финального ответа.
\end{enumerate}

\subsection{Формальные определения}
\label{subsec:formal-definitions}

\begin{definition}[Граф знаний]
    \textit{Граф знаний}~--- ориентированный мультиграф $\mathcal{G} = (V, E, R)$, где $V$~--- конечное множество вершин (сущностей), $R$~--- конечное множество типов отношений, $E \subseteq V \times R \times V$~--- множество направленных рёбер (фактов). Каждый факт представляется тройкой $(s, r, o)$, где $s \in V$~--- субъект, $r \in R$~--- отношение, $o \in V$~--- объект.
\end{definition}

\begin{definition}[Эталонный подграф рассуждения]
    Для вопроса $q$ \textit{эталонным подграфом рассуждения} называется подграф $G^* = (V^*, E^*) \subseteq \mathcal{G}$, содержащий минимальное множество троек, необходимых для логического вывода корректного ответа на $q$.
\end{definition}

\begin{definition}[Сгенерированный граф рассуждения]
    \textit{Сгенерированный граф рассуждения} $\hat{G} = (\hat{V}, \hat{E})$~--- граф, полученный путём автоматического извлечения троек из текстового ответа модели $\mathcal{M}$ на вопрос $q$.
\end{definition}

\begin{definition}[Метрики структурного соответствия]
    Для пары графов $(\hat{G}, G^*)$ определяются следующие метрики:
    \begin{itemize}
        \item \textit{Precision}: $P = \dfrac{|\hat{E} \cap E^*|}{|\hat{E}|}$~--- доля корректных рёбер среди сгенерированных;
        \item \textit{Recall}: $R = \dfrac{|\hat{E} \cap E^*|}{|E^*|}$~--- доля покрытых эталонных рёбер;
        \item \textit{F1}: $F_1 = \dfrac{2 P R}{P + R}$~--- гармоническое среднее точности и полноты;
        \item \textit{Graph Edit Distance (GED)}: минимальное число операций вставки, удаления и замены вершин и рёбер, необходимых для преобразования $\hat{G}$ в $G^*$.
    \end{itemize}
\end{definition}

\subsection{Входные и выходные данные}
\label{subsec:input-output}

\textbf{Входные данные:}
\begin{itemize}
    \item Датасет вопросов, требующих многошагового рассуждения. В работе используются два типа датасетов:
    \begin{itemize}
        \item \textit{RuleTaker}~--- синтетический датасет логических задач с явно заданными правилами вывода и фактами;
        \item \textit{Wikidata-пути}~--- вопросы, ответы на которые определяются путями длины $k$ в графе знаний Wikidata.
    \end{itemize}
    \item Граф знаний $\mathcal{G}$ (Wikidata) или набор правил и фактов (RuleTaker), из которых извлекаются эталонные подграфы $G^*$.
    \item Предобученная языковая модель $\mathcal{M}$ (Qwen3-0.6B, Llama~3.x, OLMo-Think, GPT-OSS).
\end{itemize}

\textbf{Выходные данные:}
\begin{itemize}
    \item Фреймворк smart-thinking-llm для проведения экспериментов.
    \item Количественные оценки качества рассуждений моделей по метрикам $P$, $R$, $F_1$, GED.
    \item Дообученная модель (Qwen3-0.6B), прошедшая двухэтапный пайплайн SFT~+~RL с графовой наградой.
    \item Сравнительный анализ эффективности графовой награды по отношению к бинарной оценке финального ответа.
\end{itemize}

\subsection{Требования к решению}
\label{subsec:requirements}

\subsubsection{Функциональные требования}

\begin{enumerate}
    \item \textbf{Извлечение графа из текста.} Система должна автоматически преобразовывать текстовые рассуждения модели в набор троек $(s, r, o)$, формирующих граф рассуждения $\hat{G}$.
    \item \textbf{Построение эталонных подграфов.} Для каждого вопроса из датасета должен автоматически строиться эталонный подграф $G^*$ на основе графа знаний или правил вывода.
    \item \textbf{Вычисление метрик.} Система должна вычислять метрики структурного соответствия (Precision, Recall, F1, GED) между $\hat{G}$ и $G^*$.
    \item \textbf{Поддержка нескольких датасетов.} Фреймворк должен поддерживать работу с датасетами RuleTaker и Wikidata.
    \item \textbf{Пайплайн обучения.} Система должна реализовывать двухэтапный пайплайн обучения:
    \begin{itemize}
        \item этап SFT (Supervised Fine-Tuning) для начальной адаптации модели к формату рассуждений;
        \item этап RL (Reinforcement Learning) с использованием графовой функции награды.
    \end{itemize}
    \item \textbf{Графовая функция награды.} Функция награды должна учитывать структурное соответствие сгенерированного графа эталонному, а не только корректность финального ответа.
    \item \textbf{Воспроизводимость.} Все эксперименты должны быть воспроизводимы с фиксированными гиперпараметрами и случайными затравками.
\end{enumerate}

\subsubsection{Нефункциональные требования}

\begin{enumerate}
    \item \textbf{Масштабируемость.} Фреймворк должен поддерживать модели размером от 0.6B до 7B параметров.
    \item \textbf{Модульность.} Архитектура системы должна допускать замену отдельных компонентов: модели, датасета, функции награды, алгоритма RL.
    \item \textbf{Документированность.} Код должен быть документирован и снабжён инструкциями по воспроизведению экспериментов.
\end{enumerate}

\subsection{Критерии оценки решения}
\label{subsec:criteria}

Решение считается успешным, если выполнены следующие условия:
\begin{enumerate}
    \item Разработан и функционирует фреймворк smart-thinking-llm, обеспечивающий полный цикл от генерации ответов до вычисления графовых метрик.
    \item Проведён сравнительный анализ не менее четырёх моделей на двух датасетах.
    \item Реализован двухэтапный пайплайн обучения SFT~+~RL с графовой наградой.
    \item Экспериментально продемонстрировано статистически значимое повышение метрик качества рассуждений (F1 по графовому покрытию) при использовании графовой награды по сравнению с бинарной оценкой.
\end{enumerate}
